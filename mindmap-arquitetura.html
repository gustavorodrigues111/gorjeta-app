🏗️ Arquitetura do Sistema
O AppTip é um app React de página única (~17.000 linhas) com Firebase Firestore como banco de dados. Todos os dados são carregados ao iniciar e gerenciados por uma função central handleUpdate(campo, valor).

🔥 Firebase Firestore

📱 Frontend

React 18 com hooks (sem Redux). Arquivo único App.js. CSS inline com variáveis para tema claro/escuro. Design responsivo com hook useMobile().

React + Hooks

🔥 Banco de Dados

Firebase Firestore, coleção única appdata. Cada documento tem formato { value: dados }. Cache local via IndexedDB + localStorage.

Firestore

🤖 Inteligência Artificial

API Groq com modelo Llama 3.3-70B para geração de FAQ, horários, e importação de ponto. Hook compartilhado useAiGenerate().

Groq / Llama

🔒 Segurança

Firebase App Check (reCAPTCHA v3). Login por CPF + PIN de 4 dígitos. Rate limiting: 5 tentativas → bloqueio de 30s. Persistência offline somente leitura (master).

App Check + PIN

💾 Salvamento

Função central handleUpdate(campo, valor) atualiza o estado local E salva no Firebase. Bloqueio de escrita quando offline (somente master).

handleUpdate

📊 Versionamento

Histórico de alterações em gorjetas e escalas (últimas 30 versões). Snapshots automáticos com debounce de 30s. Restauração com 1 clique.

30 versões

🔑 Flags importantes

isMaster

Flag no owner que habilita: lixeira definitiva, gerenciar admins, persistência offline com bloqueio de escrita.

isDP

Flag no manager que habilita: visualizar mensagens DP, criar sub-gestores, ver aba Caixa de notificações.

isOwner

Prop booleana passada ao RestaurantPanel. true para Owner, false para Manager. Desbloqueia funcionalidades extras.

privacyMode

Config do restaurante. Quando ativo, mascara valores de gorjeta, CPFs e mensagens DP na visão do Owner.

profile: "lider"

Perfil do manager que restringe a apenas Dashboard, Escala, Horários, Equipe e Reuniões das suas áreas.

tabsConfig / tabsGestor

Duas camadas de visibilidade: admin define o que é possível, gestor define o que o empregado vê. Ambos precisam ser true.

🧩 Componentes Principais
Todos os componentes são funções React definidas no App.js. Aqui estão os mais importantes para entender a arquitetura.
🏗️ Infraestrutura

🔧 load(key)

Carrega dados do Firestore com 3 retries. Fallback para cache localStorage se falhar.

💾 save(key, value)

Salva no Firestore + atualiza cache local. Usado internamente pelo handleUpdate.

📤 handleUpdate(campo, valor)

Função central de salvamento. Atualiza estado React + salva no Firebase. Bloqueia escrita quando offline (master).

🤖 useAiGenerate()

Hook que encapsula chamadas à API Groq (Llama 3.3-70B) com estados de loading/erro.

📱 useMobile()

Hook que retorna true se tela menor que 768px. Usado em todo o app.

📸 saveVersion()

Salva snapshot de escala ou gorjeta com debounce de 30s. Máximo 30 versões.

🗺️ Guia de Planejamento
Use esta seção para planejar novas funcionalidades. Entenda o impacto de cada tipo de mudança.
📐 Como adicionar uma nova funcionalidade

1️⃣ Novo campo de dados

Adicionar chave em const K, criar useState, incluir no load() paralelo, no objeto data, nos setters e keys do handleUpdate, e no cacheSet.

~6 pontos de edição

2️⃣ Nova aba no RestaurantPanel

Adicionar no array de abas do grupo apropriado (Operação, Pessoas, Comunicação, Config). Criar o componente da aba e o bloco {tab === "novaAba" && ...}.

~3 pontos de edição

3️⃣ Nova sub-aba no EmployeeSpreadsheet

Adicionar pill no PillBar do detalhe do empregado, criar o bloco condicional com o conteúdo da sub-aba.

~2 pontos de edição

4️⃣ Nova aba do empregado

Adicionar no array de tabs do EmployeePortal, criar componente, adicionar controle de visibilidade em tabsConfig e tabsGestor.

~4 pontos de edição

5️⃣ Novo tipo de reunião

Adicionar no array MEETING_TYPES do MeetingPlannerSection. Pode precisar de template específico de pauta.

~2 pontos de edição

6️⃣ Novo relatório PDF

Criar função que gera o PDF com jsPDF ou html2pdf. Adicionar botão de exportação no componente relevante.

⚠️ Pontos de atenção

🔴 Arquivo único

Todo o código está em App.js (~17.000 linhas). Qualquer mudança precisa cuidar para não quebrar outras partes. O build com CI=true detecta erros.

🔴 Dados globais

Todos os dados são carregados no startup. Adicionar novos campos aumenta o tempo de carregamento e o consumo do Firestore.

🟡 Permissões manuais

Não há sistema centralizado de permissões. Cada componente verifica isOwner, isMaster, isDP, perms manualmente.

🟡 Mobile vs Desktop

Algumas abas bloqueiam acesso no mobile com banner "use desktop". Considere se novas funcionalidades precisam funcionar no celular.

💡 Áreas com potencial de expansão

📊 Analytics / Dashboards

Hoje o dashboard é básico. Potencial para: gráficos de tendência, comparativo entre meses, ranking de empregados, KPIs do restaurante.

📱 App nativo / PWA

O app já é responsivo. Um PWA com notificações push seria uma evolução natural, especialmente para empregados.

🔔 Notificações em tempo real

Hoje as notificações são puxadas no login. Firebase tem suporte a listeners em tempo real (onSnapshot) que poderiam ser adicionados.

📋 Mais tipos de relatório

Relatório de gorjetas mensal, relatório de atrasos, relatório financeiro consolidado, relatório de reuniões por período.

🤖 Mais IA

Análise de padrões de atrasos, sugestão automática de feedback, previsão de custos de VT, resumo inteligente de reuniões.

🔗 Integrações

Integração com folha de pagamento, sistemas de ponto além do Sólides, WhatsApp para comunicados, calendário do Google.
